### 1. ~~精确输入排除模糊候选（预计 2-4 小时）~~✅ 已解决
**问题描述**：
- WE 共键有三种操作：点击（模糊匹配 W/E）、左滑（精确 W）、右滑（精确 E）
- 当前滑动精确输入 `ee` 时，仍会出现 `ww` 的模糊候选

**技术限制**：
- Rime 的 `speller/algebra` 的 `derive` 规则作用于词库拼音，不是用户输入
- 无论点击还是滑动，发送给引擎的都是同一个字符，无法区分输入来源
- `derive` 创建等价变体，无法区分优先级

**Lua 解决方案**：
1. **键盘层**：滑动发送大写字母（如 `E`）作为"精确输入标记"
2. **Lua Processor**（~50-80行）：
   - 拦截大写字母输入
   - 记录到"精确输入序列"
   - 转换为小写字母发送给 speller
3. **Lua Filter**（~60-100行）：
   - 读取"精确输入序列"
   - 对比候选项的拼音与用户实际输入
   - 过滤掉不精确匹配的候选（如输入 `EE` 时过滤 `ww` 候选）
4. **配置集成**：在 schema 中注册 processor 和 filter

### 3. ~~智能返回上级键盘~~ ✅ 已解决
使用 `Keyboard_last_lock` + `lock: true` 方案：
- 主键盘（default, qwerty26）设置 `lock: true`
- 次级键盘（number, edit_keyboard）设置 `lock: false`
- 返回键使用 `select: .last_lock`

详见 [Keypoint.md](Keypoint.md) 第1节。
